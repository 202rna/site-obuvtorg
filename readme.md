## 🛠 Спецификация API Маршрутов

Все эндпоинты разделены по типам HTTP-методов. Полную интерактивную документацию (Swagger) можно посмотреть локально после запуска по адресу: http://localhost:8000/docs#/

# API Documentation


### 🔐 Обозначения:
- **👤** – метод требует **авторизации** (любой залогиненный пользователь).  
- **🔒** – метод требует **роли `admin`**.

---

## Аутентификация и пользователи

| Метод | Эндпоинт | Защита | Описание |
| :--- | :--- | :---: | :--- |
| `POST` | `/api/register` | | Регистрация нового пользователя |
| `POST` | `/api/login` | | Авторизация (получение JWT-токена) |
| `GET` | `/api/me` | 👤 | Получение данных текущего пользователя |

---

## Товары (Products)

| Метод | Эндпоинт | Защита | Описание |
| :--- | :--- | :---: | :--- |
| `GET` | `/api/products` | | Получить список всех товаров (с пагинацией) |
| `POST` | `/api/products` | 👤🔒 | Создать новый товар |
| `GET` | `/api/products/{product_id}` | | Получить товар по ID |
| `PATCH` | `/api/products/{product_id}` | 👤🔒 | Частично обновить товар (цена, описание, скидка, категории, размеры) |
| `DELETE` | `/api/products/{product_id}` | 👤🔒 | Удалить товар по ID |
| `PUT` | `/api/products/{product_id}/images` | 👤🔒 | Заменить все изображения товара (загрузка новых) |

---

## 🐳 Быстрый запуск (Docker)

Проект полностью контейнеризирован. Для сборки образов и запуска всех необходимых сервисов (FastAPI, СУБД) в фоновом режиме выполните следующую команду в корневой папке проекта:

```bash
docker compose up --build -d
```

Для остановки контейнеров используйте:

```bash
docker compose down
```
