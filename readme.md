# 🚀 Документация API и запуск проекта

Краткое руководство по доступным эндпоинтам интернет-магазина и командам для развертывания через Docker.

---

## 🛠 Спецификация API Маршрутов

Все эндпоинты разделены по типам HTTP-методов. Полную интерактивную документацию (Swagger) можно посмотреть локально после запуска по адресу: http://localhost:8000/docs#/

### 📥 POST (Создание и Аутентификация)

| Маршрут | Описание | Доступ |
| :--- | :--- | :--- |
| `/api/register` | Регистрация нового аккаунта пользователя | Публичный |
| `/api/login` | Аутентификация и выдача сессионного JWT-токена | Публичный |
| `/api/products` | Добавление нового товара с загрузкой картинки | Только Админ |
| `/api/cart/{product_id}` | Добавление выбранного товара в корзину | Авторизованный |

### 📤 GET (Чтение данных)

| Маршрут | Описание | Доступ |
| :--- | :--- | :--- |
| `/api/me` | Получение закрытого профиля текущего пользователя | Авторизованный |
| `/api/products` | Просмотр витрины товаров (каталога) с пагинацией | Публичный |

### 🗑 DELETE (Удаление ресурсов)

| Маршрут | Описание | Доступ |
| :--- | :--- | :--- |
| `/api/cart` | Полная очистка корзины текущего пользователя | Авторизованный |
| `/api/products/{product_id}` | Удаление товара из БД и стирание файла с диска | Только Админ |

## 🐳 Быстрый запуск (Docker)

Проект полностью контейнеризирован. Для сборки образов и запуска всех необходимых сервисов (FastAPI, СУБД) в фоновом режиме выполните следующую команду в корневой папке проекта:

```bash
docker compose up --build -d
```

Для остановки контейнеров используйте:
```bash
docker compose down
```
